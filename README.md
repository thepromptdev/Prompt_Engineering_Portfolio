## 🚀 About Me

I am a Tester turned Prompt Engineer and AI Systems Architect specializing in building robust, production-grade LLM workflows. I treat prompt engineering as software development—moving beyond simple "chatbot interactions" to design highly constrained systems that eliminate hallucinations, enforce strict business logic, and output predictable, parseable data structures.

My primary focus is on maximizing model reliability and safety through advanced contextual framing, rigorous guardrails, and algorithmic prompting techniques.

## 🛠️ Core Competencies & Methodologies

*   **Architectural Frameworks:** CRTR (Context, Request, Training/Constraints, Response Formatting)
*   **Contextual Alignment:** Zero-Shot, Few-Shot, and Multi-Shot Prompting
*   **Reasoning Optimization:** Chain-of-Thought (CoT) and Invisible Scratchpad implementation
*   **Data Structuring:** Forcing strict JSON, XML, and Markdown array outputs
*   **Safety & Compliance:** Designing System Prompts with hard boundary constraints to protect business logic

**Techniques Used:** 
* Explicit Chain-of-Thought Mathematics
* Edge-Case Testing
* Variable Isolation

By prioritizing clarity over cleverness, I build AI systems that businesses can trust.
